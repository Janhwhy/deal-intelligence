# Processed Data

This directory contains intermediate and final processed assets generated by the deal intelligence data pipeline.

---

## Data Structure

1.  **Deal Timelines (`deals/*.json`):**
    Each JSON file represents a chronologically ordered sequence of communication events (Enron emails) and CRM events (stage transitions) corresponding to a single pseudo-deal.
2.  **Tabular Features (`tabular_features.parquet`):**
    A consolidated table containing engineered indicators (engagement touches, response latencies, stage velocity, participant count, etc.) indexed by `deal_id`.

---

## Critical Assumptions & Limitations

### 1. Synthetic Deal-Linking
The raw Enron email corpus and raw HubSpot CRM datasets are **not natively connected**. They have no shared identifiers (such as common deal IDs, client domains, or employee emails).
To bridge these sources, we implement a **deterministic synthetic linking strategy**:
- **Domain Pairing:** emails are grouped by alphabetical domain counterparts (excluding `enron.com` if other domains exist to identify external counterparts).
- **Email Clustering:** within a domain group, emails are clustered into pseudo-deals based on subject similarity (Levenshtein/Gestalt Sequence Matching) and temporal proximity (time gaps less than 7 days).
- **CRM Sampling:** each pseudo-deal is enriched by sampling an actual row-level distribution from the raw HubSpot `deals.csv`, `companies.csv`, and `contacts.csv` tables. This preserves realistic company sizes, industries, close status, and deal amounts.
- **Timeline Progression:** CRM stage changes are distributed chronologically along the email thread timeline.

### 2. Synthetic CRM Data
The HubSpot CRM dataset itself is entirely **synthetic and Faker-generated** (not real HubSpot production CRM logs). Thus, this pipeline uses a synthetic alignment heuristic to label deals derived from another dataset (the real Enron corpus), resulting in a double-layer synthetic configuration. These characteristics must be carefully considered during downstream model evaluation.
